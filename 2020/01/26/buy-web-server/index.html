0. 前言

学生党一名，阿里云、腾讯云、百度云服务器都有使用过，主要还是选择价格便宜的学生机以及一些新用户机型体验，所以希望和大家分享一下自己的购机经验。

  • 本文涉及的主要是国内的主流云服务器（国外云服务器价格一般较高，不在考虑范畴）。
  • 主要参考性价比，以1核2g1M到2核4g2M的云服务器为讨论对象。
  • 本文主要讨论的是云服务器主机。

本人目前使用的是腾讯云开发者服务器376元/2年，配置是2核4g2M，系统配置为ubuntu16.04 64x。邀请四人购购买还可以延长一年。

1. 云服务器可以用来干什么

  • 搭建个人网站或者个人博客
  • 实现一些功能，例如爬取网页，挂载游戏服务器、微信小程序等
  • 移动办公、存储
  • Linux学习

2. 云服务器对比

总览

云服务器的选择

撇开带宽，CPU+内存的组合主要分为以下几种：

  • 1核1G
  • 1核2G
  • 2核4G

1核1G不建议入手，1G内存很容易成为制约点，所以主要就是在1核2G以及2核4G之间进行取舍。我个人的建议是

  • 如果云服务器用来做搭建个人网站，或者是上手测试，用来做后台或者仓库，可以选择1核2G。一般来说，作为新手1核2G足够使用。
  • 如果云服务器需要挂载一些运算量大的进程，比如做游戏服务器，或者较大型或者较多网站，，可以选择2核4G。

带宽方面，如果业务量大的话，比如做游戏服务器，最好使用2M的服务器。对于小型网站和个人博客，如果优化得当，1M也可以使用。带宽一般较贵，本着够用就行的原则。

不同厂商的云服务器使用感受

  • 三者使用下来，阿里云服务的使用感觉最好，特别是特别定制的Workbench(类似xftp+xshell)很好用，移动端也很方便。支持SSH、VNC。
  • 其次是腾讯云。由于和微信的无缝贴合，所以移动端很方便。支持SSH、VNC。
  • 百度云个人使用体验一般，移动端使用也不是很方便。支持SSH、VNC。

3. 云服务器的配置

系统选择

建议安装Linux系统的服务器镜像，依据个人喜好，centOS或者ubuntu都可以。我个人喜欢Ubuntu多一点。

  • 因为本身服务器的资源就很有限，在2核4G的服务器下，带图形界面的Ubuntu就可以吃掉CPU20%左右的资源；
  • 其次远程连接图形界面非常卡。习惯使用命令行后，会发现会方便快捷很多，而且几乎没有延时；
  • 如果实在不习惯命令行，可以使用xftp等软件进行辅助。

总之，能不用图形界面就不用图形界面，能用Linux就不用windows。

安全组设置

安全组是用来打开和关闭云服务器网络和外界网络之间接口，可以理解为最为有效和直接的一道壁垒。一般安全组会默认全开，也就是外网可以通过所有接口访问到服务器，因此会给服务器带来很大的隐患。

个人开发使用的服务器可以让用于远程连接的22、3389端口只可以自己家庭网络所在的IP端可以访问到，打开80端口用于网站网页的外界访问即可。

具体配置可见：安全组应用案例

远程工具(Linux)

一般来说，PuTTY足够使用。也可以xftp+xshell使用，利用xftp强大的文件传输能力，也可以很快上手Linux系统。

域名

搭建网站和个人博客一定需要域名，个人建议购买.com或者.cn结尾的域名，选一个没人使用的，也很便宜。
